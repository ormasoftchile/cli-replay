## Gaps Identified for cli-replay

From Robert's analysis, seven priority improvements to support gert's runbook use case:

1. **Public pkg/ API** — Promote scenario, runner, verify packages (critical blocker)
2. **Step-level metadata** — `description`, `tags`, `id` fields
3. **Partial replay** — Start/end step control for resumable runbooks
4. **`when` conditions** — Enable conditional branching in steps
5. **Scenario composition** — `include` directive for modular runbooks
6. **Event hooks** — `on_step_start`, `on_step_complete` callbacks for observability
7. **Programmatic builder** — Fluent API for constructing scenarios in Go
