## Design Decisions

- Adapter intentionally thin (~210 LOC total) тАФ all heavy lifting stays in cli-replay engine
- Uses cli-replay's YAML format directly тАФ no format conversion overhead
- `go.mod` uses `replace` directive for local development (publication path documented)
- RecordingExecutor builds `scenario.Scenario` programmatically тАФ recorded scenarios immediately replayable

## Field Mapping Confirmed

## Scope Limitations (By Design)

- env parameter from CommandExecutor not forwarded to replay engine (replay doesn't need env for matching)
- stdin matching not implemented (gert's CommandExecutor interface doesn't pass stdin)
- Evidence collection via separate interface (separate adapter if needed)

## Key Findings

- gert's CommandExecutor interface is well-aligned with cli-replay's execution model
- Type conversion straightforward тАФ minimal impedance mismatch
- Thread-safe operation validated through concurrent test runs
- Recording path produces immediately-replayable scenarios (round-trip validated)
