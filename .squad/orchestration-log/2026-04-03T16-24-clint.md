- Analyzed gert package accessibility (pkg/ vs internal/)

## Decision Filed

**clint-gert-architecture.md** — Recommends Pattern B (transparent PATH interception) for v1 integration. Zero code coupling, zero changes to either codebase, covers primary use case of intercepting CLI tools during runbook execution. Pattern A (deep CommandExecutor integration) identified as endgame for high-value coupling.

## Key Findings

- `providers.CommandExecutor` is THE single integration seam — all CLI command execution flows through this interface
- Only stdio-mode tool steps and legacy cli steps interceptable via CommandExecutor
- gert's replay is simpler but less powerful than cli-replay (no wildcards, regexes, budgets, unordered groups, stdin matching)
- gert exports pkg/ so can import cli-replay if cli-replay promotes internals to pkg/
- JSON-RPC and MCP tools have their own process management, unreachable by CommandExecutor pattern
