## Key Findings

- cli-replay's internal/ boundary is the critical blocker: ALL functional packages are internal, only cmd exports public API
- gert runs scenarios in three modes: `plan` (dry-run), `apply` (real execution), `verify` (evidence collection)
- CLI wrapper pattern works well due to structured JSON output from `cli-replay exec`
- Partial replay and step metadata are needed for real-world runbook coverage scenarios
- Concurrency model differs: gert has parallel iterate and parallel steps; cli-replay assumes sequential
- Conditional steps (`when` field) exist in both tools but are unused in cli-replay
