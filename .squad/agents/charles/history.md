### 2026-04-03 — Capture & Serialization Pipeline Analysis

#### Architecture: Dual-Mode Binary
- `main.go` uses invocation name to decide mode: `cli-replay` → Cobra management commands; anything else (e.g., `kubectl`) → intercept mode calling `runner.ExecuteReplay()`.
- Intercept mode reads `CLI_REPLAY_SCENARIO` env var to find scenario file, matches `os.Args` against expected steps, returns canned responses.

#### Command Execution (Record Path)
- **Direct mode** (no `--command` filters): `recorder.Execute()` runs `exec.Command(args[0], args[1:]...)`, tees stdout/stderr to both caller and internal buffers, logs one `RecordedCommand`. Does NOT capture stdin.
- **Shim mode** (`--command` filters): `recorder.SetupShims()` creates per-command shim scripts in a temp dir. `recorder.Execute()` prepends shim dir to PATH, injects `CLI_REPLAY_RECORDING_LOG` and `CLI_REPLAY_SHIM_DIR` env vars, then runs via `platform.WrapCommand()`.
- `platform.WrapCommand()` is OS-specific: Unix uses `bash -c`, Windows uses `powershell.exe -NoProfile -ExecutionPolicy Bypass -Command`.
- Shim scripts (Unix = bash, Windows = .cmd+.ps1) capture argv, stdout, stderr, exit code, timestamp, and stdin (if piped/redirected) into JSONL.

#### Command Execution (Replay Path)
- `cmd/exec.go` directly uses `exec.Command(childArgv[0], childArgv[1:]...)` — no custom wrapper.
- Child env built by `runner.BuildChildEnv()`: prepends intercept dir to PATH, sets `CLI_REPLAY_SESSION` and `CLI_REPLAY_SCENARIO`.
- Platform-specific signal handling: Unix uses process groups (`Setpgid=true`) with SIGINT/SIGTERM forwarding + SIGKILL fallback; Windows uses Job Objects with `CREATE_SUSPENDED` start.
- `cmd/run.go` only sets up the intercept environment and emits shell setup commands — actual replay happens when shims are called by the user's script.

#### Capture Mechanics — What Gets Captured
- **Always captured**: argv, stdout, stderr, exit code, timestamp (RFC3339).
- **Conditionally captured**: stdin (only in shim mode, only if piped/redirected).
- **NOT captured**: environment variables (filtered at replay time via `meta.vars` + `meta.security.deny_env_vars`), wall-clock duration (only timestamp per entry).
- Stdout/stderr are base64-encoded in JSONL if non-UTF-8.

#### Serialization Formats
1. **Recording log** (`.jsonl`): One JSON object per line. Fields: `timestamp`, `argv`, `exit`, `stdout`, `stderr`. Written by shim scripts during recording.
2. **Scenario file** (`.yaml`): Strict YAML with `meta` + `steps`. Loaded via `yaml.Decoder.KnownFields(true)`. Generated by `recorder.ConvertToScenario()` + `recorder.WriteYAMLFile()`.
3. **State file** (`.json`): Tracks replay progress. Fields: step counts, current step, active group, captures, intercept dir, last updated, scenario hash. Written atomically (temp file + rename) by `runner.WriteState()`.
4. **JSON Schema**: `schema/scenario.schema.json` (draft-07) defines the scenario format with `additionalProperties: false` everywhere.

#### Scenario YAML Structure
```yaml
meta:
  name: string (required)
  description: string
  vars: {key: value}  # templated via {{ .key }}
  security:
    allowed_commands: [string]
    deny_env_vars: [glob-pattern]  # e.g., AWS_*, SECRET_*
  session:
    ttl: duration-string  # e.g., 10m
steps:
  - match:
      argv: [string]  # supports {{ .any }}, {{ .regex "..." }}
      stdin: string    # optional exact match
    respond:
      exit: int        # 0-255, default 0
      stdout: string   # mutually exclusive with stdout_file
      stderr: string   # mutually exclusive with stderr_file
      stdout_file: path
      stderr_file: path
      delay: duration-string
      capture: {key: value}
    calls:
      min: int
      max: int
    when: string
  - group:
      mode: unordered  # only supported mode
      name: string
      steps: [leaf-steps]  # no nesting
```

#### Playback Fidelity
- **Matching**: Element-by-element argv comparison. Supports literal, `{{ .any }}` wildcard, `{{ .regex "..." }}`. Lengths must match exactly.
- **Stdin validation**: Optional `match.stdin` for exact stdin content matching. Shim reads stdin if piped.
- **Ordering**: Steps are consumed sequentially by default. `group.mode: unordered` allows any-order matching within a group (barrier semantics).
- **Call bounds**: `calls.min`/`calls.max` control repeat invocations per step. Verify checks minimums; max limits progression.
- **Timing**: `respond.delay` is supported but recordings only capture timestamps, not durations. No automatic timing replay.
- **Error conditions**: Exit codes preserved (0-255). Signal-killed processes map to `128+signum` on Unix.
- **Template rendering**: Captures chain across steps via `{{ .capture.key }}`. Env vars merge into `meta.vars` keys only (filtered by deny patterns).

#### File I/O Patterns
- Recording log: written to temp dir during recording, read by `recorder.Finalize()`.
- Scenario YAML: read by `scenario.LoadFile()` (strict), written by `recorder.WriteYAMLFile()`.
- State files: stored in `.cli-replay/` relative to CWD. Path derived from scenario path hash + session ID via `runner.StateFilePathWithSession()`. Atomic writes.
- Intercept dirs: `.cli-replay/intercept-*` containing symlinks (Unix) or .cmd wrappers (Windows).
- Cleanup: `cmd/clean.go` handles state + intercept removal. TTL-based expiry. Recursive mode skips `.git`, `node_modules`, `vendor`, `.terraform`, `__pycache__`.

#### Platform Concerns
- **Build tags**: `unix.go` (`//go:build !windows`), `windows.go` (`//go:build windows`), `jobobject_windows.go`.
- **Shim generation**: Unix = single bash script; Windows = `.cmd` entrypoint + `.ps1` companion.
- **Shell wrapping**: Unix = `bash -c`; Windows = `powershell.exe -NoProfile -ExecutionPolicy Bypass -Command`.
- **Intercept creation**: Unix = symlink to cli-replay binary; Windows = `.cmd` wrapper or copy.
- **PATH filtering**: OS-specific separator (`:` vs `;`) and case-insensitive on Windows.
- **Process management**: Unix = process groups; Windows = Job Objects with suspended start.
- **Signal forwarding**: Unix = SIGINT/SIGTERM to group, fallback SIGKILL; Windows = `os.Interrupt`, fallback `Process.Kill()`.

#### Key File Paths
- `main.go` — entry point, dual-mode dispatch
- `cmd/exec.go` — exec command (setup + child + verify + cleanup)
- `cmd/record.go` — record command (direct or shim capture)
- `cmd/run.go` — run command (session setup, shell emission)
- `cmd/exec_unix.go` / `cmd/exec_windows.go` — OS-specific signal/process handling

- `internal/runner/` — state management, replay execution, env building
- `internal/scenario/` — YAML loading/validation, data model
- `internal/matcher/` — argv pattern matching (literal, wildcard, regex)
- `internal/template/` — Go text/template rendering with captures
- `internal/platform/` — shim generation, shell wrapping, intercept creation, Job Objects
- `internal/envfilter/` — env var deny-list filtering
- `internal/verify/` — verification result building, JSON/JUnit formatting
- `schema/scenario.schema.json` — JSON Schema (draft-07)
- `testdata/recordings/` — JSONL fixtures
- `testdata/scenarios/` — YAML scenario fixtures
- `testdata/fixtures/` — raw output fixtures
